## Conciliação Emusys — novos tipos de divergência + filtros/paginação/ações (BRT)

Contexto: a verificação contra a API do Emusys (CG) achou inconsistências de classificação/valor que precisam de decisão humana, não correção automática. Levadas para a aba de Conciliação (fila), com a UI ganhando filtros, paginação e mais ações.

### Backend — `sync-matriculas-emusys` (deploy v5, dry-run)
- Motor `reconciliar()` refatorado: retorna `{ upd (AUTO), divergencias[] }` — antes era 1 coisa por aluno (early-return). Agora AUTO e fila coexistem; vários tipos por aluno.
- **`valor_divergente`**: quando a fonte é ambígua (`liquido < 0`/`cheio <= 0` = `desconto_fixo` embutido no `valor_mensalidade`), **não** auto-aplica; enfileira com sugestão `efetivo = cheio − cond`. Só enfileira se `efetivo > 0` (bolsista integral com mensal 0/fixo cheio → efetivo 0 → ignora, não-pagante legítimo). Bug corrigido: motor antigo aplicaria `valor_parcela` negativo.
- **`classificacao_divergente`**: bolsista→REGULAR só quando paga o **cheio integral** (`liquido === cheio`, sem desconto); REGULAR→bolsista quando API `bolsa=true` (INT se efetivo≤0, PARC se >0). O flag `bolsa` da API **não é confiável** sozinho.
- **Dedup**: `(aluno|tipo)` já com decisão humana não é reenfileirado (respeita a decisão).
- Popular (dry-run, 3 unidades): CG `classificacao_divergente=4` (Adriana/Lavynea/Olivia→REGULAR; Sofia→BOLSISTA_PARC, paga ~190), `valor_divergente=12`; Recreio `valor_divergente=2`; Barra 0. **Nada escrito em `alunos`** (dry-run).
